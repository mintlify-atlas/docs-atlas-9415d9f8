## Customer Accounts

Every customer has an account that gives them access to the client portal.

### Account Information

**Core Fields:**
- **First Name**: Customer's first name
- **Last Name**: Customer's last name
- **Email**: Login credential and contact email (must be unique)
- **Password**: Securely hashed, never stored in plain text

**Additional Features:**
- **Avatar**: Automatically generated from Gravatar based on email
- **Email Verification**: Status of email confirmation
- **Two-Factor Authentication**: Optional 2FA for enhanced security
- **Role**: Admin role assignment (for staff members)

### Customer Registration

Customers can register through the public registration page:

<Steps>
  <Step title="Navigate to Registration">
    Access `/register` on your Paymenter installation
  </Step>
  
  <Step title="Fill Out Form">
    Provide first name, last name, email, and password
  </Step>
  
  <Step title="Email Verification">
    System sends verification email (if enabled in settings)
  </Step>
  
  <Step title="Complete Profile">
    Customer can add additional information like address and company details
  </Step>
  
  <Step title="Start Shopping">
    Browse products and place orders
  </Step>
</Steps>

<Info>
You can disable public registration and manually create customer accounts if you prefer a private registration process.
</Info>

## Custom Properties

Custom properties allow you to collect additional information from customers beyond the standard fields.

### Common Use Cases

- **Billing Address**: Street, city, state/province, postal code, country
- **Company Information**: Company name, registration number
- **Tax Details**: VAT number, tax ID, business type
- **Phone Numbers**: Primary, mobile, support contact
- **Custom Fields**: Industry, referral source, special requirements

### Creating Custom Properties

<Steps>
  <Step title="Navigate to Custom Properties">
    Go to **Administration** → **Custom Properties**
  </Step>
  
  <Step title="Create New Property">
    Click **Create** and configure:
    - **Name**: Display label (e.g., "Company Name")
    - **Key**: Internal identifier (e.g., "company_name")
    - **Type**: Text, number, dropdown, checkbox, etc.
    - **Required**: Whether customers must fill it out
    - **Show on Invoice**: Display this property on invoices
    - **Show on Registration**: Ask for this during signup
  </Step>
  
  <Step title="Property Values">
    Customers can fill out properties in their account settings
  </Step>
</Steps>

<Tip>
Use the "Show on Invoice" option for properties like billing address and tax ID that should appear on official invoices.
</Tip>

### Property Types

- **Text**: Single-line text input
- **Textarea**: Multi-line text input
- **Number**: Numeric values
- **Select**: Dropdown with predefined options
- **Checkbox**: Yes/no boolean
- **Date**: Date picker

### Creating Customers Manually

Admins can create customer accounts:

1. Navigate to **Administration** → **Users** → **Create**
2. Fill in required fields (first name, last name, email, password)
3. Optionally assign admin role
4. Save customer
5. Add custom properties if needed

<Note>
Manually created customers receive a welcome email with their account details. You can disable this in settings.
</Note>

## Customer Communication

Paymenter sends automated emails to customers for various events:

### Email Notifications

- **Account Creation**: Welcome email with login instructions
- **Email Verification**: Confirmation link
- **Password Reset**: Secure reset link
- **Order Confirmation**: Details of new orders
- **Invoice Generated**: New invoice notification with payment link
- **Payment Received**: Payment confirmation
- **Service Activated**: Service ready notification
- **Service Expiring Soon**: Renewal reminder
- **Service Suspended**: Service suspended due to non-payment
- **Ticket Updates**: Replies to support tickets

### Email Templates

Customize email templates:

1. Navigate to **Administration** → **Notification Templates**
2. Select template to edit
3. Modify subject and body
4. Use variables for dynamic content (customer name, invoice amount, etc.)
5. Preview and save

<Tip>
Use clear, professional language in email templates. Include relevant links and contact information for customer support.
</Tip>

## Customer Notification Preferences

Customers can control which notifications they receive:

1. Navigate to **Account** → **Notifications**
2. Toggle notification types:
   - Email notifications
   - Browser push notifications
   - Specific event types (invoices, services, tickets)
3. Save preferences

<Info>
Critical notifications (password reset, payment confirmation) cannot be disabled to prevent account security issues.
</Info>

## Security Features

### Two-Factor Authentication (2FA)

Customers can enable 2FA for enhanced account security:

<Steps>
  <Step title="Enable 2FA">
    Navigate to **Account** → **Security** → **Enable Two-Factor Authentication**
  </Step>
  
  <Step title="Scan QR Code">
    Use authenticator app (Google Authenticator, Authy) to scan QR code
  </Step>
  
  <Step title="Enter Verification Code">
    Confirm setup by entering generated code
  </Step>
  
  <Step title="Save Recovery Codes">
    Store recovery codes in safe place for account recovery
  </Step>
</Steps>

<Warning>
If customer loses access to 2FA device and recovery codes, admin intervention is required to disable 2FA and regain access.
</Warning>

### Password Security

- Passwords are hashed using bcrypt (never stored in plain text)
- Password reset via email verification
- Session management with timeout
- Failed login attempt tracking

### Admin Tools

**Disable 2FA:**
If customer loses access, admin can disable 2FA from customer edit page

**Reset Password:**
Admin can set new password for customer (customer receives notification)

**Session Management:**
View and terminate active customer sessions

## Customer Analytics

Track important customer metrics:

### Key Metrics

- **Total Customers**: Active account count
- **New Customers**: Registrations over time period
- **Active Customers**: Customers with active services
- **Customer Lifetime Value (CLV)**: Total revenue per customer
- **Churn Rate**: Percentage of customers cancelling

### Customer Segments

Group customers by:

- **Service Count**: Single service vs. multiple services
- **Total Spend**: High-value vs. low-value customers
- **Service Types**: Product categories purchased
- **Geographic Location**: By country or region
- **Payment Behavior**: On-time payers vs. frequently late

<Tip>
Use customer segments for targeted marketing campaigns, custom pricing, or priority support tiers.
</Tip>

## Troubleshooting

### Customer Can't Login

- Verify email address is correct
- Check if account is verified (if verification required)
- Try password reset process
- Check for failed login attempts (rate limiting)
- Verify 2FA codes if enabled

### Customer Not Receiving Emails

- Check spam/junk folder
- Verify email address in account is correct
- Test email configuration from admin panel
- Review email logs for delivery errors
- Check email server reputation (blacklists)

### Customer Properties Not Showing

- Verify property is marked as "show on registration" or customer must add manually
- Check property visibility settings
- Ensure customer has saved property values
