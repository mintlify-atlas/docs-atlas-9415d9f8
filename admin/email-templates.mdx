Paymenter's notification system allows you to customize email and in-app notifications sent to customers. Each notification template can be individually configured with custom content, delivery settings, and user preferences.

## Overview

Access notification templates at **Admin Panel → Other → Notification Templates** or visit `/admin/notification-templates`.

## Notification Template List

The template list displays:

- **Subject** - Email subject line (searchable)
- **Enabled** - Whether the notification is active
- Click any template to edit its configuration

## Template Structure

Each notification template has:

- **Key** - Unique identifier (cannot be changed after creation)
- **Name** - Human-readable template name
- **Enabled** - Master toggle for the entire notification
- **Email Template** - Email subject, body, and recipients
- **In-App Notification** - Push notification title and body
- **User Preference Settings** - Control options for end users

**Source**: `app/Models/NotificationTemplate.php:13`

## Email Configuration

The Email Template section configures email notifications.

### Email Fields

<Accordion title="Subject">
  **Required field** - Email subject line
  
  - Maximum 255 characters
  - Supports template variables (e.g., `Your invoice #{invoice.number} is ready`)
  - Plain text only
</Accordion>

<Accordion title="Mail Enabled">
  Controls email delivery and user preferences:
  
  **Options**:
  - **User Choice, Default On** - User can disable, enabled by default
  - **User Choice, Default Off** - User can enable, disabled by default
  - **Force On** - Always send, user cannot disable
  - **Force Off** - Never send, user cannot enable
  
  **Source**: `app/Admin/Resources/NotificationTemplateResource.php:58`
</Accordion>

<Accordion title="Body">
  **Required field** - Email content
  
  - Supports Markdown and HTML
  - Toolbar buttons disabled (use raw Markdown/HTML)
  - Supports template variables
  - Applied to email layout (header/footer from [Settings](/admin/settings))
  
  **Example**:
  ```markdown
  Hello {{ $user->first_name }},
  
  Your order #{{ $order->id }} has been completed.
  
  Thank you for your business!
  ```
</Accordion>

<Accordion title="CC Recipients">
  Carbon copy recipients (optional)
  
  - Multiple email addresses allowed
  - Enter one email per tag
  - Validated as email addresses
  - Example use: Always CC billing@company.com on invoice emails
</Accordion>

<Accordion title="BCC Recipients">
  Blind carbon copy recipients (optional)
  
  - Multiple email addresses allowed
  - Recipients hidden from other recipients
  - Validated as email addresses
  - Example use: Archive all notifications to archive@company.com
</Accordion>

**Source**: `app/Admin/Resources/NotificationTemplateResource.php:48`

## In-App Configuration

The In-App Notification section configures push notifications displayed in the application.

### In-App Fields

<Accordion title="In-App Title">
  **Required field** - Notification title shown in app
  
  - Maximum 255 characters
  - Supports template variables
  - Should be concise (displays in notification list)
  - Disabled when In-App Enabled is set to "Force Off"
</Accordion>

<Accordion title="In-App Enabled">
  Controls in-app delivery and user preferences:
  
  **Options**:
  - **User Choice, Default On** - User can disable, enabled by default
  - **User Choice, Default Off** - User can enable, disabled by default  
  - **Force On** - Always show, user cannot disable
  - **Force Off** - Never show, user cannot enable
  
  **Source**: `app/Admin/Resources/NotificationTemplateResource.php:89`
</Accordion>

<Accordion title="In-App Body">
  **Required field** - Notification message content
  
  - Plain text (no Markdown)
  - Supports template variables
  - Displays when notification is expanded
  - Disabled when In-App Enabled is set to "Force Off"
</Accordion>

<Accordion title="In-App URL">
  Optional link when user clicks notification
  
  - Maximum 255 characters
  - Supports Laravel route helpers with variables
  - Example: `{{ route('invoices.show', $invoice) }}`
  - Disabled when In-App Enabled is set to "Force Off"
</Accordion>

**Source**: `app/Admin/Resources/NotificationTemplateResource.php:78`

## User Preference Control

<Accordion title="Edit Preference Message">
  **Optional field** - Message shown to users when editing their notification preferences
  
  - Explains what the notification is for
  - Helps users decide whether to enable/disable
  - Example: "Notifies you when a new invoice is generated"
  - Only relevant when Mail/In-App Enabled is set to "User Choice"
</Accordion>

### How User Preferences Work

The `NotificationEnabledStatus` enum controls notification delivery:

1. **Force** - Always send (ignores user preference)
2. **Never** - Never send (ignores user preference)
3. **ChoiceOn** - User can control, default enabled
4. **ChoiceOff** - User can control, default disabled

**Source**: `app/Models/NotificationTemplate.php:32`

### Permission Checking

The `isEnabledForPreference()` method determines if a notification should be sent:

```php
$template->isEnabledForPreference($userPreference, 'mail');
$template->isEnabledForPreference($userPreference, 'app');
```

**Logic**:
- If status is `Force`, always returns `true`
- If status is `Never`, always returns `false`
- If status is `ChoiceOn/Off`, checks user's preference
- Falls back to default if no user preference exists

**Source**: `app/Models/NotificationTemplate.php:52`

## Template Variables

Notification templates support dynamic variables using Laravel's Blade syntax.

### Common Variables

<Accordion title="User Variables">
  Available in all notifications:
  
  - `{{ $user->first_name }}` - User's first name
  - `{{ $user->last_name }}` - User's last name
  - `{{ $user->email }}` - User's email address
  - `{{ $user->name }}` - Full name (first + last)
</Accordion>

<Accordion title="Invoice Variables">
  Available in invoice-related notifications:
  
  - `{{ $invoice->id }}` - Invoice ID
  - `{{ $invoice->number }}` - Formatted invoice number
  - `{{ $invoice->total }}` - Invoice total amount
  - `{{ $invoice->due_date }}` - Payment due date
  - `{{ $invoice->status }}` - Invoice status
</Accordion>

<Accordion title="Service Variables">
  Available in service-related notifications:
  
  - `{{ $service->id }}` - Service ID
  - `{{ $service->name }}` - Service name
  - `{{ $service->status }}` - Service status
  - `{{ $service->product->name }}` - Product name
</Accordion>

<Accordion title="Order Variables">
  Available in order-related notifications:
  
  - `{{ $order->id }}` - Order ID
  - `{{ $order->status }}` - Order status
  - `{{ $order->total }}` - Order total
</Accordion>

<Accordion title="Ticket Variables">
  Available in ticket notifications:
  
  - `{{ $ticket->id }}` - Ticket ID
  - `{{ $ticket->subject }}` - Ticket subject
  - `{{ $ticket->status }}` - Ticket status
  - `{{ $ticket->department }}` - Department name
</Accordion>

### Using Variables

<Info>
Variables use Laravel Blade syntax. The available variables depend on the notification context. Check your notification implementation to see which variables are passed.
</Info>

**Example Email Body**:
```markdown
Hello {{ $user->first_name }},

Your invoice **{{ $invoice->number }}** for {{ $invoice->total }} is now due.

Due Date: {{ $invoice->due_date->format('F j, Y') }}

[View Invoice]({{ route('invoices.show', $invoice) }})

Thank you!
```

## Notification Preferences

The `NotificationPreference` model stores user-specific notification settings:

```php
$template->preferences()  // All user preferences for this template
```

**Fields**:
- `notification_template_id` - Template reference
- `user_id` - User who owns the preference
- `mail_enabled` - User's email preference
- `in_app_enabled` - User's in-app preference

**Source**: `app/Models/NotificationTemplate.php:36`

## Troubleshooting

<Accordion title="Emails Not Sending">
  1. Check that **Mail Disable** is turned off in [Settings](/admin/settings)
  2. Verify SMTP configuration is correct
  3. Ensure template is enabled
  4. Check **Mail Enabled** is not set to "Force Off" or "Never"
  5. Verify user hasn't disabled the notification in their preferences
</Accordion>

<Accordion title="Template Variables Not Working">
  1. Verify variable syntax: `{{ $variable->property }}`
  2. Ensure the variable exists in the notification context
  3. Check for typos in variable names
  4. Review notification implementation code for available variables
</Accordion>

<Accordion title="In-App Notifications Not Appearing">
  1. Check **In-App Enabled** is not set to "Never"
  2. Verify user has enabled the notification in preferences
  3. Ensure template is enabled
  4. Check browser console for JavaScript errors
</Accordion>
