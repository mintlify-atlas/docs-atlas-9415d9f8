## Overview

Access tax rate management at **Admin Panel → Configuration → Tax Rates** or visit `/admin/tax-rates`.

## Enabling Tax

Before configuring tax rates, enable the tax system:

<Steps>
  <Step title="Navigate to Settings">
    Go to **Admin → System → Settings** and open the **Tax** tab
  </Step>
  
  <Step title="Enable Tax">
    Toggle **Tax Enabled** to `ON`
    
    **Setting**: `tax_enabled`
  </Step>
  
  <Step title="Choose Tax Type">
    Select how tax is calculated:
    
    - **Inclusive** - Prices include tax (tax is extracted)
    - **Exclusive** - Prices exclude tax (tax is added)
    
    **Setting**: `tax_type`
  </Step>
  
  <Step title="Save Settings">
    Click **Save** to enable the tax system
  </Step>
</Steps>

<Info>
After enabling tax, the Tax Rates menu item will appear in the admin navigation.
</Info>

## Tax Calculation Types

### Inclusive Tax

<Accordion title="How Inclusive Tax Works">
  Tax is **included** in the displayed price.
  
  **Example**: Product price is $100 with 20% tax
  - Displayed price: **$100.00**
  - Net price (excluding tax): $83.33
  - Tax amount: $16.67
  - Total paid: $100.00
  
  **Use case**: Europe, Australia, and regions where prices must include tax by law
</Accordion>

### Exclusive Tax

<Accordion title="How Exclusive Tax Works">
  Tax is **added** to the displayed price.
  
  **Example**: Product price is $100 with 20% tax
  - Displayed price: **$100.00**
  - Net price: $100.00
  - Tax amount: $20.00
  - Total paid: $120.00
  
  **Use case**: United States, Canada, and regions where tax is added at checkout
</Accordion>

<Warning>
Changing tax type affects how all prices are displayed and calculated. Consider the impact on existing products and orders before changing this setting.
</Warning>

## Creating Tax Rates

<Steps>
  <Step title="Navigate to Tax Rates">
    Go to **Admin → Configuration → Tax Rates** and click **New Tax Rate**
  </Step>
  
  <Step title="Enter Tax Details">
    Fill in the required fields:
    
    **Name**: Descriptive name (e.g., "UK VAT", "California Sales Tax")
    - Required
    - Maximum 255 characters
    - For internal reference
  </Step>
  
  <Step title="Set Tax Rate">
    Enter the percentage rate
    
    **Rate**: Tax percentage (e.g., `20` for 20%, `7.5` for 7.5%)
    - Required
    - Decimal values supported (up to 2 decimal places)
    - Do not include the % symbol
  </Step>
  
  <Step title="Select Country">
    Choose which country this rate applies to:
    
    **Country**: Country or "All Countries"
    - Required
    - Must be unique (one rate per country)
    - Select "All Countries" for global default
  </Step>
  
  <Step title="Save Tax Rate">
    Click **Create** to save the tax rate
  </Step>
</Steps>

## Tax Rate Fields

<Accordion title="Name">
  **Field**: `name`
  
  Descriptive name for the tax rate.
  
  - **Required**: Yes
  - **Max Length**: 255 characters
  - **Examples**: "VAT 20%", "GST", "Sales Tax", "Standard Rate"
  - **Purpose**: Helps identify the rate in admin panel and reports
</Accordion>

<Accordion title="Rate">
  **Field**: `rate`
  
  Tax percentage as a decimal number.
  
  - **Required**: Yes
  - **Format**: Decimal number (e.g., 20, 7.5, 19.25)
  - **Precision**: Up to 2 decimal places
  - **Display**: Shows with % suffix in UI
  - **Validation**: Numeric value required
</Accordion>

<Accordion title="Country">
  **Field**: `country`
  
  Country code this rate applies to.
  
  - **Required**: Yes
  - **Options**: Country list from `config('app.countries')` + "All Countries"
  - **Unique**: Only one rate per country
  - **Special value**: `all` for global/default rate
  - **Format**: Two-letter country code (e.g., `US`, `GB`, `DE`)
</Accordion>

## Tax Calculation Logic

Paymenter determines which tax rate to apply based on customer location:

### Priority System

1. **Country-Specific Rate** - If a rate exists for the customer's country, use it
2. **Global Rate** - If no country-specific rate exists, use the "All Countries" rate
3. **No Tax** - If neither exists, no tax is applied (0%)

**Source**: `app/Classes/Settings.php:662`

### Customer Country Detection

The customer's country is determined from:

1. User properties (custom field `country`)
2. Billing address country
3. Account registration country

<Info>
Country values are normalized to two-letter country codes (ISO 3166-1 alpha-2) before lookup.
</Info>

### Tax Calculation Query

```php
TaxRate::whereIn('country', [$customerCountry, 'all'])
    ->orderByRaw('country = ? desc', [$customerCountry])
    ->first();
```

This query:
- Searches for both country-specific and global rates
- Prioritizes country-specific over global
- Returns the most specific applicable rate

## Editing Tax Rates

Click any tax rate in the list to edit it.

<Warning>
**Considerations when editing tax rates:**

- Changes apply to new orders immediately
- Existing unpaid invoices may need manual adjustment
- Historical invoices are not retroactively changed
- Consider creating a new rate instead of editing for audit purposes
</Warning>

## Deleting Tax Rates

Tax rates can be deleted using bulk actions:

1. Select one or more tax rates using checkboxes
2. Choose **Delete** from the bulk actions menu
3. Confirm deletion

<Warning>
Deleting a tax rate does not affect historical invoices, but may cause issues if referenced by active pricing rules. Verify no active products depend on the rate before deletion.
</Warning>

## Tax Exemptions

Paymenter does not currently support per-customer tax exemptions at the model level. To implement tax exemptions:

<Steps>
  <Step title="Identify Exempt Customers">
    Mark customers as tax-exempt using custom user properties or tags
  </Step>
  
  <Step title="Manual Override">
    Manually adjust invoices for tax-exempt customers
  </Step>
  
  <Step title="Custom Development">
    Implement custom tax exemption logic in your Paymenter installation
  </Step>
</Steps>

<Info>
Consider extending Paymenter with custom tax exemption functionality if you frequently serve tax-exempt customers (e.g., non-profits, government entities).
</Info>

## Troubleshooting

<Accordion title="Tax Not Calculating">
  1. Verify **Tax Enabled** is turned on in Settings
  2. Check that a tax rate exists for the customer's country or "All Countries"
  3. Confirm customer has a country set in their profile
  4. Ensure products are not marked as tax-exempt (if your system supports it)
</Accordion>

<Accordion title="Wrong Tax Rate Applied">
  1. Check the customer's country in their user properties
  2. Verify the tax rate for that country is correct
  3. Check if a more specific rate exists that overrides the expected one
  4. Review the tax priority system (country-specific > global)
</Accordion>

<Accordion title="Cannot Create Tax Rate">
  1. Ensure the country hasn't already been used (one rate per country)
  2. Verify Tax is enabled in Settings (menu won't show if disabled)
  3. Check you have the `admin.tax-rates.create` permission
</Accordion>

<Accordion title="Tax Shows Incorrectly on Invoice">
  1. Verify the **Tax Type** setting (Inclusive vs Exclusive)
  2. Check that the correct tax rate was applied at order time
  3. Confirm invoice template is displaying tax correctly
  4. Review invoice calculation logic in your theme
</Accordion>

## Technical Details

### Tax Model

The `TaxRate` model is simple:

```php
protected $fillable = ['name', 'rate', 'country'];
```

**Database Table**: `tax_rates`

**Relationships**: None (standalone model)

**Source**: `app/Models/TaxRate.php:12`

### Settings Integration

Tax rates are accessed via the `Settings::tax()` helper:

```php
$taxRate = Settings::tax($user);  // Returns TaxRate model or 0
```

This method:
- Accepts optional user parameter (defaults to current user)
- Reads user's country from properties
- Queries for applicable tax rate
- Returns `TaxRate` model or `0` if no tax
- Uses Laravel's `once()` helper to cache result per request

**Source**: `app/Classes/Settings.php:662`
