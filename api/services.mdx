The Services API allows you to create, retrieve, update, and delete customer services. Services represent active instances of products that customers have purchased.

## The Service Object

<ResponseField name="id" type="integer">
  Unique identifier for the service
</ResponseField>

<ResponseField name="quantity" type="integer">
  Number of units of this service
</ResponseField>

<ResponseField name="price" type="decimal">
  Price per billing period
</ResponseField>

<ResponseField name="status" type="string">
  Service status: `pending`, `active`, `cancelled`, `suspended`
</ResponseField>

<ResponseField name="currency_code" type="string">
  Three-letter currency code (e.g., USD, EUR)
</ResponseField>

<ResponseField name="expires_at" type="date">
  Next renewal date (null for one-time or free services)
</ResponseField>

<ResponseField name="created_at" type="datetime">
  Timestamp when the service was created
</ResponseField>

<ResponseField name="updated_at" type="datetime">
  Timestamp when the service was last updated
</ResponseField>

### Relationships

<ResponseField name="user" type="object">
  The customer who owns this service
</ResponseField>

<ResponseField name="product" type="object">
  The product this service is an instance of
</ResponseField>

<ResponseField name="order" type="object">
  The order this service was created from
</ResponseField>

<ResponseField name="coupon" type="object">
  Applied coupon/discount code (if any)
</ResponseField>

<ResponseField name="invoices" type="array">
  Invoices associated with this service
</ResponseField>

<ResponseField name="properties" type="array">
  Custom properties and configuration options
</ResponseField>

## List Services

<CodeGroup>
```bash cURL
curl https://your-domain.com/api/v1/admin/services \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Accept: application/json"
```

```javascript JavaScript
const response = await fetch('https://your-domain.com/api/v1/admin/services', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Accept': 'application/json'
  }
});

const services = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://your-domain.com/api/v1/admin/services',
    headers={
        'Authorization': 'Bearer YOUR_API_KEY',
        'Accept': 'application/json'
    }
)

services = response.json()
```

```php PHP
$response = Http::withHeaders([
    'Authorization' => 'Bearer ' . $apiKey,
    'Accept' => 'application/json',
])->get('https://your-domain.com/api/v1/admin/services');

```
</CodeGroup>

  Filter services by price

## Create a Service

<CodeGroup>
```bash cURL
curl -X POST https://your-domain.com/api/v1/admin/services \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": 1,
    "product_id": 1,
    "plan_id": 1,
    "quantity": 1,
    "price": "9.99",
    "status": "active",
    "currency_code": "USD"
  }'
```

```javascript JavaScript
const response = await fetch('https://your-domain.com/api/v1/admin/services', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Accept': 'application/json',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    user_id: 1,
    product_id: 1,
    plan_id: 1,
    quantity: 1,
    price: '9.99',
    status: 'active',
    currency_code: 'USD'
  })
});

const service = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://your-domain.com/api/v1/admin/services',
    headers={
        'Authorization': 'Bearer YOUR_API_KEY',
        'Accept': 'application/json',
        'Content-Type': 'application/json'
    },
    json={
        'user_id': 1,
        'product_id': 1,
        'plan_id': 1,
        'quantity': 1,
        'price': '9.99',
        'status': 'active',
        'currency_code': 'USD'
    }
)

service = response.json()
```
</CodeGroup>

<ParamField body="plan_id" type="integer" required>

## Service Status

Services have four possible statuses:

### pending
Service has been created but is not yet active. This may be waiting for:
- Payment confirmation
- Manual approval
- Provisioning to complete

### active
Service is fully functional and in use. Recurring services will generate renewal invoices.

### suspended
Service is temporarily disabled, usually due to:
- Non-payment
- Terms of service violation
- Manual suspension by administrator

### cancelled
Service has been terminated and will not renew. The service may remain accessible until the expiration date.

## Service Lifecycle

Typical service lifecycle:

1. **Created** - Service created with `pending` status
2. **Activated** - Status changed to `active` once payment is confirmed
3. **Renewed** - For recurring services, `expires_at` is updated on each payment
4. **Suspended** - If payment fails, status changes to `suspended`
5. **Reactivated** - Once payment is received, status returns to `active`
6. **Cancelled** - User or admin cancels the service

## Filtering Active Services

Get all active services:

```bash
curl "https://your-domain.com/api/v1/admin/services?filter[status]=active" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Finding Expiring Services

Get services expiring soon (requires custom date filtering in your application):

```bash
curl "https://your-domain.com/api/v1/admin/services?sort=expires_at" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

<Note>
  Services are automatically renewed by Paymenter's cron jobs when `expires_at` is reached. Ensure your cron is properly configured.
</Note>
