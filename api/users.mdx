---
title: Users
description: Manage customer accounts via the API
---

The Users API allows you to create, retrieve, update, and delete customer accounts in Paymenter.

## The User Object

<ResponseField name="id" type="integer">
  Unique identifier for the user
</ResponseField>

<ResponseField name="first_name" type="string">
  User's first name
</ResponseField>

<ResponseField name="last_name" type="string">
  User's last name
</ResponseField>

<ResponseField name="email" type="string">
  User's email address (must be unique)
</ResponseField>

<ResponseField name="email_verified_at" type="datetime">
  Timestamp when the email was verified (null if not verified)
</ResponseField>

<ResponseField name="created_at" type="datetime">
  Timestamp when the user was created
</ResponseField>

<ResponseField name="updated_at" type="datetime">
  Timestamp when the user was last updated
</ResponseField>

### Relationships

<ResponseField name="role" type="object">
  The user's role (for admin users)
</ResponseField>

<ResponseField name="properties" type="array">
  Custom properties associated with the user
</ResponseField>

<ResponseField name="orders" type="array">
  User's orders
</ResponseField>

<ResponseField name="services" type="array">
  User's active services
</ResponseField>

<ResponseField name="invoices" type="array">
  User's invoices
</ResponseField>

<ResponseField name="tickets" type="array">
  User's support tickets
</ResponseField>

<ResponseField name="credits" type="array">
  User's account credits
</ResponseField>

## List Users

<CodeGroup>
```bash cURL
curl https://your-domain.com/api/v1/admin/users \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Accept: application/json"
```

```javascript JavaScript
const response = await fetch('https://your-domain.com/api/v1/admin/users', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Accept': 'application/json'
  }
});

const data = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://your-domain.com/api/v1/admin/users',
    headers={
        'Authorization': 'Bearer YOUR_API_KEY',
        'Accept': 'application/json'
    }
)

users = response.json()
```

```php PHP
$response = Http::withHeaders([
    'Authorization' => 'Bearer ' . $apiKey,
    'Accept' => 'application/json',
])->get('https://your-domain.com/api/v1/admin/users');

$users = $response->json();
```
</CodeGroup>

### Query Parameters

<ParamField query="per_page" type="integer" default="15">
  Number of users to return per page (max 100)
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number for pagination
</ParamField>

<ParamField query="filter[first_name]" type="string">
  Filter users by first name
</ParamField>

<ParamField query="filter[last_name]" type="string">
  Filter users by last name
</ParamField>

<ParamField query="filter[email]" type="string">
  Filter users by email address
</ParamField>

<ParamField query="sort" type="string">
  Sort users by field. Available: `id`, `first_name`, `last_name`, `email`, `created_at`. Prefix with `-` for descending.
</ParamField>

<ParamField query="include" type="string">
  Include related resources. Available: `properties`, `orders`, `services`, `invoices`, `tickets`, `credits`, `role`. Comma-separated.
</ParamField>

## Create a User

<CodeGroup>
```bash cURL
curl -X POST https://your-domain.com/api/v1/admin/users \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "newuser@example.com",
    "password": "secure_password_123",
    "first_name": "Jane",
    "last_name": "Smith"
  }'
```

```javascript JavaScript
const response = await fetch('https://your-domain.com/api/v1/admin/users', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Accept': 'application/json',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'newuser@example.com',
    password: 'secure_password_123',
    first_name: 'Jane',
    last_name: 'Smith'
  })
});

const user = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://your-domain.com/api/v1/admin/users',
    headers={
        'Authorization': 'Bearer YOUR_API_KEY',
        'Accept': 'application/json',
        'Content-Type': 'application/json'
    },
    json={
        'email': 'newuser@example.com',
        'password': 'secure_password_123',
        'first_name': 'Jane',
        'last_name': 'Smith'
    }
)

user = response.json()
```
</CodeGroup>

### Request Body

<ParamField body="email" type="string" required>
  User's email address (must be unique)
</ParamField>

<ParamField body="password" type="string" required>
  User's password (minimum 8 characters)
</ParamField>

<ParamField body="first_name" type="string">
  User's first name (max 255 characters)
</ParamField>

<ParamField body="last_name" type="string">
  User's last name (max 255 characters)
</ParamField>

<ParamField body="email_verified_at" type="datetime">
  Timestamp for email verification (ISO 8601 format)
</ParamField>

<ParamField body="role_id" type="integer">
  ID of the role to assign (for admin users)
</ParamField>

### Response

```json
{
  "data": {
    "type": "users",
    "id": "42",
    "attributes": {
      "first_name": "Jane",
      "last_name": "Smith",
      "email": "newuser@example.com",
      "email_verified_at": null,
      "created_at": "2024-01-20T15:45:00.000000Z",
      "updated_at": "2024-01-20T15:45:00.000000Z"
    }
  }
}
```

## Delete a User

<CodeGroup>
```bash cURL
curl -X DELETE https://your-domain.com/api/v1/admin/users/1 \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Accept: application/json"
```

```javascript JavaScript
const response = await fetch('https://your-domain.com/api/v1/admin/users/1', {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Accept': 'application/json'
  }
});

// Returns 204 No Content on success
```

```python Python
import requests

response = requests.delete(
    'https://your-domain.com/api/v1/admin/users/1',
    headers={
        'Authorization': 'Bearer YOUR_API_KEY',
        'Accept': 'application/json'
    }
)

# Returns 204 No Content on success
```
</CodeGroup>

### Path Parameters

<ParamField path="id" type="integer" required>
  The ID of the user to delete
</ParamField>

### Response

Returns HTTP 204 No Content on success.

<Warning>
  Deleting a user is permanent and cannot be undone. All associated data (orders, services, invoices) will also be affected.
</Warning>

## User Properties

Users can have custom properties stored as key-value pairs. When including properties in API responses, they appear in the `meta` section:

```json User with Properties
{
  "data": {
    "type": "users",
    "id": "1",
    "attributes": { ... },
    "relationships": {
      "properties": { "data": [...] }
    }
  },
  "meta": {
    "properties": {
      "company": "Acme Corp",
      "phone": "+1234567890",
      "address": "123 Main St"
    }
  }
}
```
