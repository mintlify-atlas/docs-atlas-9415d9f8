The Invoices API allows you to create, retrieve, update, and delete customer invoices. Invoices represent amounts owed by customers for services.

## The Invoice Object

<ResponseField name="id" type="integer">
  Unique identifier for the invoice
</ResponseField>

<ResponseField name="status" type="string">
  Invoice status: `pending`, `paid`, `cancelled`
</ResponseField>

<ResponseField name="currency_code" type="string">
  Three-letter currency code (e.g., USD, EUR)
</ResponseField>

<ResponseField name="due_at" type="date">
  Due date for payment
</ResponseField>

<ResponseField name="created_at" type="datetime">
  Timestamp when the invoice was created
</ResponseField>

<ResponseField name="updated_at" type="datetime">
  Timestamp when the invoice was last updated
</ResponseField>

### Relationships

<ResponseField name="user" type="object">
  The customer who owes this invoice
</ResponseField>

<ResponseField name="items" type="array">
  Line items on the invoice
  <Expandable title="Invoice Item object">
    <ResponseField name="id" type="integer">
      Item ID
    </ResponseField>
    <ResponseField name="description" type="string">
      Item description
    </ResponseField>
    <ResponseField name="quantity" type="integer">
      Quantity of items
    </ResponseField>
    <ResponseField name="price" type="decimal">
      Price per item
    </ResponseField>
  </Expandable>
</ResponseField>

## List Invoices

<CodeGroup>
```bash cURL
curl https://your-domain.com/api/v1/admin/invoices \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Accept: application/json"
```

```javascript JavaScript
const response = await fetch('https://your-domain.com/api/v1/admin/invoices', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Accept': 'application/json'
  }
});

const invoices = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://your-domain.com/api/v1/admin/invoices',
    headers={
        'Authorization': 'Bearer YOUR_API_KEY',
        'Accept': 'application/json'
    }
)

invoices = response.json()
```

```php PHP

    'Authorization' => 'Bearer ' . $apiKey,
    'Accept' => 'application/json',
])->get('https://your-domain.com/api/v1/admin/invoices');

```
</CodeGroup>

### Query Parameters

<ParamField query="per_page" type="integer" default="15">
  Number of invoices to return per page (max 100)
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number for pagination
</ParamField>

<ParamField query="filter[id]" type="integer">
  Filter invoices by ID
</ParamField>

<ParamField query="filter[currency_code]" type="string">
  Filter invoices by currency code
</ParamField>

<ParamField query="filter[user_id]" type="integer">

<ParamField query="filter[status]" type="string">

<ParamField query="include" type="string">

## Create an Invoice

<CodeGroup>
```bash cURL
curl -X POST https://your-domain.com/api/v1/admin/invoices \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Accept: application/json" \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": 1,
    "currency_code": "USD",
    "status": "pending",
    "due_at": "2024-02-15"
  }'
```

```javascript JavaScript
const response = await fetch('https://your-domain.com/api/v1/admin/invoices', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Accept': 'application/json',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    user_id: 1,
    currency_code: 'USD',
    status: 'pending',
    due_at: '2024-02-15'
  })
});

const invoice = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://your-domain.com/api/v1/admin/invoices',
    headers={
        'Authorization': 'Bearer YOUR_API_KEY',
        'Accept': 'application/json',
        'Content-Type': 'application/json'
    },
    json={
        'user_id': 1,
        'currency_code': 'USD',
        'status': 'pending',
        'due_at': '2024-02-15'
    }
)

invoice = response.json()
```
</CodeGroup>

### Request Body

<ParamField body="user_id" type="integer" required>
  ID of the user who owes this invoice
</ParamField>

<ParamField body="currency_code" type="string" required>
  Three-letter currency code (e.g., USD, EUR, GBP)
</ParamField>

<ParamField body="status" type="string" required>
  Invoice status: `pending`, `paid`, `cancelled`
</ParamField>

<ParamField body="due_at" type="date">
  Due date for payment (ISO 8601 format: YYYY-MM-DD)
</ParamField>

<ParamField body="number" type="string">
  Custom invoice number (auto-generated if not provided)
</ParamField>

<Note>
  Creating an invoice creates an empty invoice. You'll need to add invoice items separately using the Invoice Items API endpoint.
</Note>

### Response


## Invoice Status

Invoices have three possible statuses:

### pending
Invoice has been created but not yet paid. Customers can view and pay pending invoices.

### paid
Invoice has been fully paid. Payment has been received and processed.

### cancelled
Invoice has been cancelled and is no longer payable. Used for voided or invalidated invoices.

## Filtering Invoices

### Get Pending Invoices

```bash
curl "https://your-domain.com/api/v1/admin/invoices?filter[status]=pending" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Get User's Invoices

```bash
curl "https://your-domain.com/api/v1/admin/invoices?filter[user_id]=1" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Get Overdue Invoices

To find overdue invoices, filter by pending status and sort by due date:

```bash
curl "https://your-domain.com/api/v1/admin/invoices?filter[status]=pending&sort=due_at" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

<Note>
  You'll need to check the `due_at` date in your application to determine which are actually overdue.
</Note>

## Invoice Lifecycle

Typical invoice workflow:

1. **Created** - Invoice is created with `pending` status
2. **Items Added** - Line items are added for services/products
3. **Email Sent** - Invoice notification email sent to customer
4. **Payment Received** - Customer pays via payment gateway
5. **Marked Paid** - Status updated to `paid` automatically
6. **Services Activated** - Associated services are activated

## Invoice Calculations

Invoices automatically calculate totals based on line items:

- **Subtotal**: Sum of (quantity × price) for all items
- **Tax**: Applied based on user's location and tax settings
- **Total**: Subtotal + Tax
- **Remaining**: Total - Paid amount (from transactions)

<Note>
  Tax calculations and invoice snapshots are handled automatically by Paymenter based on your tax configuration.
</Note>
